관찰자 패턴
  데이터는 공급자가 관리하고 관찰자는
  이를 구독하여 데이터를 얻는 방식

공급자 - 데이터 보관, 변경 공급
관찰자 - 데이터 소비

특정 컴포넌트가 공급자에 data를 보관
관찰자는 프로퍼티를 대대손손 내려받지 말고
공급자의 데이터를 관찰해서 소비

이런 흐름을 만들어 주는게 컨텍스트
상위 컴포넌트에 공급자를 구현
하위 컴포넌트에서 이를 직접 구독

-------------------------

- 소비자는 공급자보다 낮은 계층에 존재
- 소비자는 공급자가 제공하는 콜백으로 데이터 변경 가능
  data down, action up ( DDAU )

이런 규칙을 지켜서 단방향 데이터 흐름을 만들어
데이터의 일관성을 유지한다.
